ধরো, তুমি একটি রেস্টুরেন্টের ম্যানেজার, আর এই রেস্টুরেন্টটাই হলো একটি JavaScript প্রোগ্রাম। রেস্টুরেন্টের **ডাইনিং রুম** হলো **গ্লোবাল এক্সিকিউশন কনটেক্সট**—যেখানে সবকিছু শুরু হয়: টেবিল, মেনু, ওয়েটার—একদম যেমন JS-এ গ্লোবাল ভেরিয়েবল, ফাংশন, অবজেক্ট থাকে।

একজন **কাস্টমার অর্ডার দিলে**, ওয়েটার সেটা **কিচেনে** নিয়ে যায়—এই অর্ডার নেওয়া এবং কিচেনে যাওয়া হলো **নতুন এক্সিকিউশন কনটেক্সট** তৈরি হওয়া।

**কিচেনে**, শেফ নিজের টুলস আর ইনগ্রিডিয়েন্ট দিয়ে খাবার প্রস্তুত করে—একদম যেভাবে JS-এ প্রতিটি ফাংশনের আলাদা ভেরিয়েবল আর ফাংশন থাকে।

খাবার তৈরি হলে সেটা আবার কাস্টমারের কাছে পৌঁছে দেওয়া হয়—এটা হলো **ফাংশন থেকে রিটার্ন হওয়া ভ্যালু** গ্লোবাল কনটেক্সটে ফিরে যাওয়া।

এইভাবে, যেমন রেস্টুরেন্টে একাধিক কাজ আলাদা জায়গায় হয়, তেমনি JavaScript-এও একাধিক **execution context**-এর মাধ্যমে পুরো প্রোগ্রাম চলে।
